CivilRightsSummarizedAI

Built a multi-document legal case summarization pipeline for the Civil Rights Litigation Clearinghouse using ingestion, evaluation, and fine-tuning workflows. The project emphasizes reproducibility, human-in-the-loop QA, and the tradeoffs involved in using LLM systems for high-stakes legal-document summarization.

Python NLP LLM Evaluation LoRA Fine-Tuning SQLite Testing
More details

Problem

Civil-rights case materials often span multiple documents and require a reviewer to piece together procedural history, parties, and outcomes across a large record. The project goal was to create a pipeline that could ingest case materials, generate summaries, and evaluate quality in a way that supported careful human review rather than replacing it.

Approach

I worked from an end-to-end pipeline perspective: ingestion, storage, preparation, model experimentation, and evaluation. The public repo includes installable source code, tests, fixture data, reporting artifacts, and QA-oriented tools so the system can be reviewed as a complete engineering project rather than just a modeling notebook.

Results & Impact

The project produced a documented comparison between checkpoints and showed that the last training checkpoint was not the strongest one. That outcome reinforced an important lesson for applied AI systems: evaluation and review workflow matter as much as model generation, and higher-stakes domains need quality gates instead of blind automation.

AWS Test: NYC 311 Service Request Analysis

Built an AWS-based workflow for analyzing a large sample of NYC 311 service requests using S3, Athena, and SageMaker. The project focuses on reproducible cloud structure, documented SQL analysis, and preparation for later modeling work on complaint-resolution patterns.

AWS S3 Athena SageMaker SQL Python Data Documentation
More details

Problem

Cloud analytics work can become difficult to review when data paths, table definitions, and analysis questions are only captured in class notebooks. This project was designed to turn that workflow into a documented public repository with clear dataset context, analysis questions, and Athena-ready queries.

Approach

I organized the project around the full workflow: documenting the dataset, defining the S3 and Athena structure, writing stakeholder- oriented SQL, and outlining a simple modeling export for SageMaker. The repository now includes data notes, query files, sanity checks, and a concise stakeholder-summary draft.

Results & Impact

The strongest outcome here is the structure itself: the repo now reads as a cloud-analysis project instead of a loose set of class exercises. It shows how I think about traceability, query design, and preparing raw service-request data for both reporting and downstream modeling.

Predicting League of Legends Outcomes

Developed an applied machine-learning project to predict ten-minute League of Legends match outcomes from ranked-match telemetry. The repo combines reproducible preprocessing, exploratory analysis, engineered features, and comparisons across baseline and nonlinear models.

Python TensorFlow XGBoost pandas Jupyter Notebooks Feature Engineering
More details

Problem

Match-state data at the ten-minute mark captures early objective control, economy, vision, and combat information, but it is not always obvious which features matter most for predicting eventual outcomes. The project goal was to build a clean experimental setup for comparing predictive approaches on that structured telemetry.

Approach

I used a reproducible preprocessing pipeline to generate train-test splits, summary artifacts, and engineered feature exports. From there, the repo supports multiple modeling paths, including logistic regression, neural networks, random forests, and gradient-boosted trees, with supporting figures and metrics stored in version control.

Results & Impact

The project demonstrates a full applied-ML workflow rather than a one-off notebook: data loading, EDA, feature engineering, baseline performance measurement, and model comparison. It highlights my interest in building experiments that are inspectable and easy for teammates or instructors to rerun.
